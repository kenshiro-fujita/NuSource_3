# Product Brief: NuSource_3

## Executive Summary

チームや組織は常に変化にさらされている——全社的な方針転換、新メンバーの合流、リモート化。これらの局面では本音の対話が不可欠だが、声を上げずに距離を置く人がいたり、表面的な合意に留まったりする。ワークショップや対話セッションはこれを解消する有力な手段だが、既存のリモートツールでは共在感が薄く、XRコラボツールは「道具箱」に留まっている。

本プロダクトは、対話の体験をデザインし、AIシミュレーションで事前検証し、ARグラスで共在感のある対話を実現し、その内容を自動的にグラフィックレコーディングとして記録する一気通貫のプラットフォームである。チームの関係性の変容を通じた創造的コラボレーションの実現を目指し、Nulabのミッション「To make creating simple and enjoyable」に直結する。

## Core Vision

### Problem Statement

チームや組織は常に変化にさらされている——新メンバーの合流、部署間の連携、組織の方針転換、リモート化による相互理解の希薄化。こうした局面では、メンバー同士が本音で対話し、互いの考えや感情を理解することが不可欠である。しかし実際には、変化に対して声を上げずに距離を置く人がいたり、表面的な合意にとどまって本質的な理解に至らなかったりする。

ワークショップやチームでの対話セッションはこれらを解消する有力な手段だが、既存のリモートツール（Zoom、Miro等）では「同じ場にいる感覚」が薄く、腹を割った話し合いが生まれにくい。一方、XR/ARコラボツール（Glue、Campfire 3D等）は「道具箱」に留まり、対話の体験そのものをデザインする発想がない。

### Problem Impact

- **変化への適応が遅れる**: 組織の方針転換時に本音の対話がないまま進むと、消極的に離脱するメンバーが生まれ、変化への対応力や施策の実行力が大幅に低下する
- **対話の「場」を作るコストが高い**: 効果的な対話には対面が必要とされ、出張コストやスケジュール調整がボトルネックになり、実施頻度が限られる
- **設計段階で効果が検証できない**: 対話セッションの設計者は、本番まで体験の効果がわからない。頭の中だけで設計するため、実際の参加者の反応と乖離しやすい
- **対話の成果が消える**: どれだけ良い対話が生まれても、記録が議事録テキストに留まり、対話の温度感や関係性の変化が保存されない。効果の持続・共有が困難

### Why Existing Solutions Fall Short

- **Zoom / Teams / Meet**: リモートでの対話は可能だが、共在感が薄く身体性がない。腹を割った話し合いが生まれにくい
- **Miro / FigJam**: 2Dホワイトボードとして優秀だが、「対話の体験」を設計する機能はない
- **Glue / MeetinVR / Spatial**: VR空間で共在感は出せるが「道具箱」でしかない。対話の流れをデザインする思想がない
- **Campfire 3D**: 3Dモデルの身体的操作に強いが設計レビュー特化で、チームの対話や関係性の文脈がない
- **全ツール共通の欠落**: 対話セッションの設計者は、設計→検証→実施→記録を別々のツールで断片的に行っており、一気通貫で支援するプラットフォームが存在しない

### Proposed Solution

ARグラスを活用した、チームの対話体験のデザイン・実施・記録の一気通貫プラットフォーム。ARグラスは体験を完成させるレイヤーであり、プロダクトの入口ではない。体験デザインとAIシミュレーションはWeb/デスクトップで完結し、ARグラスは本番セッションで共在感を実現するために使用する。

**ユーザージャーニー:**
課題の定義 → 対話セッションの適切性判断 → 体験設計の補助 → AIシミュレーションによる試運転 → 本番セッション（ARグラス装着）

**コア機能:**

1. **体験デザインエンジン**: 対話の流れ（問い→対話→ワーク→振り返り等）を設計するツール。ワークショップに限らず、チームの本音を静かにぶつけ合うような話し合い全般に対応。体験設計のプロセス自体が教育コンテンツとして機能し、ワークショップデザインの民主化を促進する
2. **AIシミュレーション**: 設計した体験をAIに仮想参加者として参加させ、想定したリアクションの検証を実施できる
3. **AR共在空間**: 参加者がARグラスを通じて同じ空間に「いる」感覚で対話に参加する。身体性を活かした体験が可能
4. **自動グラフィックレコーディング**: 対話の内容・流れ・温度感が自動的に視覚的に記録・可視化される。普段のMTGでも単独で利用可能

**利用シーンのグラデーション:**

| 頻度 | 利用シーン | 使用機能 |
|------|-----------|---------|
| 日常 | 普段のMTG | 自動グラレコ＋AR(チームにより自由に) |
| 週次〜月次 | チーム対話セッション | 体験デザイン＋グラレコ |
| 四半期〜 | ワークショップ | フル機能（AR含む） |

**開発フェーズ（想定）:**

- Phase 1（MVP）: 体験デザインエンジン＋自動グラレコ（Web）
- Phase 2: AIシミュレーション統合
- Phase 3: AR共在空間

### Key Differentiators

- **「道具箱」ではなく「対話の体験デザイン」**: 既存ツールが付箋やホワイトボードを提供するのに対し、対話体験全体のデザインを支援する
- **AIシミュレーション**: 対話セッションの事前テスト走行は市場に存在しない機能
- **対話の自動グラレコ化**: 人が意図的に記録する必要がなく、対話の温度感まで含めて視覚的に残る。普段のMTGでも単独利用でき、日常的な接点を作る
- **ワークショップデザインの民主化**: 体験設計のプロセス自体が教育コンテンツとして機能し、専門スキルがなくても対話の場を設計できるようになる。使うこと自体が学びになるツール
- **関係性の変容が目的**: 効率や生産性ではなく、人と人・人と概念の関係性を変えることを設計思想の中心に据えている
- **ARグラスは体験の完成レイヤー**: VRの没入型ではなく、日常の延長で使えるAR体験。プロダクトの入口ではなく、本番セッションで共在感を最大化するために使用する
- **日常→非日常のグラデーション**: 普段のMTGでの自動グラレコから、ワークショップでのフル機能利用まで、利用頻度に応じた段階的な体験を提供する
- **市場タイミング**: ARグラス主流化元年（2026年）、VR会議室モデルの限界証明（Horizon Workrooms終了）
- **開発者自身が最初の顧客**: ワークショップデザインのドメイン知識を持つ開発者が、自分のペインを解決する
- **Nulab顧客基盤の活用**: Backlog利用の14,000チーム以上の既存顧客基盤に対し、チームコラボレーションの延長線上としてリーチ可能
